## Done Checklist (Must All Be True)

### A) Canonical MLS Path + Fail-Closed Guarantees

- [x] Real OpenMLS path is active in canonical send/read flow (`create_group -> create_application_message -> process_incoming`).
- [x] No plaintext fallback when encryption is required.
- [x] Pending proposals/commit state blocks app-message send.
- [x] Lifecycle tamper defenses are fail-closed:
  - invalid pending operation rejected
  - stage/merge epoch regression rejected
  - partial snapshot payload rejected
  - out-of-order clear/merge rejected
- [x] Concurrency races are fail-closed with deterministic outcome (single winner, explicit loser errors).

### B) Key Lifecycle Hardening (Not Done Yet)

- [x] Key package persistence policy implemented (create, consume, replenish thresholds).
  - `conversation_security_client_inventories`
  - `Famichat.Chat.ConversationSecurityClientInventoryStore`
  - `Famichat.Chat.ConversationSecurityKeyPackagePolicy` (planned rename target: `Famichat.Chat.ConversationSecurityClientInventoryPolicy`)
- [x] Rotation policy implemented and tested (scheduled + trigger-based rotation).
- [x] Rejoin/recovery durability implemented and tested (state loss + deterministic recovery).
- [ ] Revocation strategy implemented and tested (device/user revocation effects on MLS state).
- [x] Key-lifecycle events are observable via telemetry without leaking sensitive data.

### C) Operational Rollout Gates (Not Done Yet)

- [ ] Canonical flow timing capture is automated and checked against declared budget.
- [ ] MLS failure-rate telemetry gate is defined (threshold + block condition) and enforced.
- [ ] Coverage snapshot for MLS-critical suites is captured and tracked.
- [ ] `docs:boundary-check` is green and naming/ownership drift is blocked in lint flow.
