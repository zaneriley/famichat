## Loop Model

Run these phases in order for every MLS increment:

1. **Build**: implement smallest production-path change with tests.
2. **Break**: red-team with adversarial cases and negative-path checks.
3. **Harden**: patch discovered failures, tighten contracts, rerun gates.
4. **Record**: update docs/status with what failed, what was fixed, and what remains.

No parallel LLM-only path is allowed. Frontend/API/CLI/agent flows must exercise the same backend contracts.

## Iteration 1 (Completed): Runtime Health Fail-Closed Gate

### What was added

1. Message send/read MLS-enforced paths now check `MLS.nif_health/0` before crypto operations.
2. MLS runtime is treated as ready only when health status is explicitly `"ok"`/`"healthy"`.
3. Degraded runtime now fails closed with `:unsupported_capability` and reason `:mls_runtime_not_ready`.
4. New contract tests assert send/read fail closed when runtime health is degraded.

### Why this matters

Prevents false confidence from placeholder/deterministic NIF behavior (`openmls_not_wired`) being treated as secure encryption when enforcement is enabled.

## Iteration 2 (Completed): Real OpenMLS Vertical Slice + Adversarial Contract Cases

### What was added

1. Rust NIF now runs a real OpenMLS-backed in-memory two-member group flow per `group_id`:
   - `create_group` provisions sender/recipient sessions via Welcome join path.
   - `create_application_message` encrypts through sender session.
   - `process_incoming` decrypts through recipient session.
   - `process_incoming` now supports message-id scoped read idempotency cache so canonical read paths can re-read stored ciphertext deterministically.
   - message-id cache includes ciphertext-integrity checks (mismatch => `:storage_inconsistent`).
   - Session snapshot export/restore is now implemented:
     - Snapshot fields are emitted in operation payloads (`session_sender_storage`, `session_recipient_storage`, signer blobs, cache blob).
     - `create_group`, `create_application_message`, `process_incoming`, and `export_group_info` can restore runtime state from provided snapshot fields.
2. Health contract updated from placeholder semantics to real runtime readiness:
   - `nif_health` now reports `status: "ok", reason: "openmls_ready"` when OpenMLS bootstrap checks pass.
3. Adversarial MLS contract tests added:
   - malformed ciphertext rejection
   - cross-group ciphertext misuse rejection
   - replay rejection on second processing attempt
   - message-id/ciphertext aliasing rejection
   - snapshot round-trip restore behavior validation
4. Rust and Elixir contract suites are green after this integration:
   - `./run rust:check`
   - MLS contract-focused ExUnit suites
5. Canonical MessageService path now has a real-NIF characterization test proving:
   - ciphertext is stored at rest
   - repeated reads remain stable (no replay-related read regression)
   - persisted conversation security state can recover decryption after runtime state reset

### Why this matters

This moves MLS from scaffold semantics to a real cryptographic execution path and validates fail-closed outcomes under active misuse attempts, not just happy-path behavior.

## Iteration 3 (Completed): Lifecycle Tamper + Ordering Hardening

### What was added

1. `ConversationSecurityLifecycle` stage/merge paths now fail-close when lifecycle metadata is tampered:
   - rejects regressive staged-epoch payloads during stage operations.
   - rejects invalid/unknown pending operation values.
   - rejects stale/invalid `staged_epoch` values (must advance past current epoch).
   - rejects regressive merge-epoch payloads.
   - rejects partial snapshot fragments that do not satisfy full snapshot integrity shape.
2. Lifecycle adversarial tests now cover:
   - out-of-order `clear_pending_commit -> merge_pending_commit` sequence rejection.
   - tampered pending operation rejection.
   - tampered/stale staged epoch rejection.
   - merge-epoch regression rejection.
   - partial snapshot payload tampering rejection.
   - concurrent `stage_pending_commit` race behavior (single-winner guarantee, fail-closed losers).
   - concurrent `merge_pending_commit` race behavior (single-winner guarantee, fail-closed losers).
3. Messaging contract tests continue to assert send-path fail-closed behavior while pending commits are unresolved (`:pending_proposals`).

### Why this matters

This closes a real class of lifecycle abuse/regression where malformed metadata or race conditions could otherwise permit ambiguous state transitions, snapshot-shape drift, or epoch regression.
