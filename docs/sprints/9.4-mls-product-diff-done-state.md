## Primary Done Gate (User-Minimum P0)

All items must be true to call this done:

### 1) Encrypted messaging feels normal
- [x] Users can send/receive encrypted direct and group messages without obvious UX regressions.

### 2) Revoked devices are actually cut off
- [x] A revoked device is immediately blocked from protected send/read paths for new conversation activity.
- [x] The blocked device sees an explicit reason (for example `device revoked`), not a silent failure.

### 3) Non-revoked users/devices keep working
- [x] Healthy participants continue messaging with no interruption caused by revocation of others.

### 4) Recovery/rejoin is reliable
- [x] A recovered/new device can rejoin and continue safely with no forked or corrupted conversation security state.

### 5) Security failures are explicit to humans
- [x] Users receive actionable states (`device revoked`, `recovery required`) where relevant.
- [x] Required-encryption paths never silently degrade.

### 6) Proven by repeatable end-to-end behavior
- [x] The above is proven by repeatable multi-user, multi-device end-to-end runs, not only unit tests.

## Required E2E Proof

At minimum, record repeated successful runs of:

1. User A/B exchange encrypted messages in direct and group conversations.
2. Revoke one of User A's devices.
3. Verify revoked device cannot continue protected conversation activity and receives explicit state.
4. Verify User A other device and User B remain unaffected.
5. Rejoin/recover revoked/lost device.
6. Verify post-recovery messaging works and state remains consistent under retries.

Evidence should include exact commands/tests executed and pass/fail results.
